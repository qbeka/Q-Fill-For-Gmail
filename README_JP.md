# Gmail コード自動入力 拡張機能

[🇬🇧 English](README.md) | 🇯🇵 日本語

Gmailの受信トレイから認証コードを自動的に読み取り、Webフォームに入力するChrome拡張機能です。Gmail APIを読み取り専用OAuthで使用し、すべての処理をローカルで行い、データを保存・送信することはありません。

---

## ✨ 機能

- 🔐 最新のGmailメッセージから4〜8桁のコードを自動抽出
- ⚡ アクティブなタブのコード入力欄に即座に自動入力
- 🖱️ ポップアップUIから手動でメールチェック可能
- 🔒 ローカルのみで処理（サーバーや外部ストレージなし）
- 🛡️ Gmail APIを読み取り専用OAuth権限で使用
- 🎨 美しいモダンなUIと視覚的フィードバック

---

## 🔒 セキュリティとプライバシー

- ✅ Gmailは`gmail.readonly`スコープのOAuthでアクセス
- ✅ メールやコードを保存、送信、ログに記録することはありません
- ✅ コード抽出とフォーム自動入力は完全にローカルマシンで実行
- ✅ 外部サービスやテレメトリは一切なし
- ✅ クライアントIDはローカルに保存され、gitにコミットされません

この拡張機能はデータ収集ツールでは**ありません**。ユーザーのプライバシーを最優先に、一般的なタスクを自動化するだけです。

## 📦 セットアップとインストール

### 前提条件

1. **Google Cloudプロジェクト** - Google CloudプロジェクトとOAuth認証情報が必要です
2. **Chrome ブラウザ** - この拡張機能はGoogle Chrome (v88以降) で動作します

### ステップ 1: Google Cloudのセットアップ

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 新しいプロジェクトを作成（または既存のプロジェクトを選択）
3. **Gmail API**を有効化:
   - 「APIとサービス」>「ライブラリ」に移動
   - 「Gmail API」を検索
   - 「有効にする」をクリック
4. OAuth 2.0認証情報を作成:
   - 「APIとサービス」>「認証情報」に移動
   - 「認証情報を作成」>「OAuth クライアントID」をクリック
   - アプリケーションタイプとして「Chromeアプリ」を選択
   - 拡張機能ID を追加（拡張機能を読み込んだ後に取得）
   - **クライアントID**をコピー

### ステップ 2: 拡張機能のセットアップ

1. **このリポジトリをクローンまたはダウンロード**:
   ```bash
   git clone https://github.com/qbeka/Code_Autofill_Extension_KANA.git
   cd Code_Autofill_Extension_KANA
   ```

2. **拡張機能を設定**:
   ```bash
   # テンプレートからconfig.jsを作成
   cp config/config.template.js config/config.js
   ```

3. **`config/config.js`を編集**し、`YOUR_CLIENT_ID_HERE`を実際のGoogle OAuth クライアントIDに置き換えます:
   ```javascript
   const CONFIG = {
     CLIENT_ID: 'your-actual-client-id.apps.googleusercontent.com',
     // ... その他の設定
   };
   ```

4. **`manifest.json`を更新**してクライアントIDを設定:
   - `manifest.json`を開く
   - `oauth2.client_id`フィールドを見つける
   - クライアントIDに置き換える

### ステップ 3: Chromeで拡張機能を読み込む

1. Chromeを開き、`chrome://extensions/` にアクセス
2. **デベロッパーモード**を有効化（右上のトグル）
3. **「パッケージ化されていない拡張機能を読み込む」**をクリック
4. 拡張機能ディレクトリを選択
5. 表示される**拡張機能ID**をコピー
6. Google Cloud Console > 認証情報に戻る
7. OAuthクライアントを編集して拡張機能IDを追加

### ステップ 4: 拡張機能をテスト

1. Chromeで拡張機能アイコンをクリック
2. 「Gmailに接続」をクリック
3. 拡張機能を承認
4. 新しいタブで`test-form.html`を開く
5. Gmailにテストメールを送信:
   ```
   件名: テスト認証コード
   本文: あなたの認証コードは: 123456
   ```
6. 拡張機能ポップアップで「今すぐメールをチェック」をクリック
7. コードがテストフォームに自動入力されるはずです！

## 🔧 トラブルシューティング

### 問題: 拡張機能がGmailに接続できない

**解決策:**
1. Google Cloud Consoleで Gmail API が有効になっているか確認
2. OAuth クライアント ID が正しく設定されているか確認
3. `config/config.js` と `manifest.json` の両方でクライアントIDが一致しているか確認
4. 拡張機能IDがGoogle Cloud Consoleに正しく登録されているか確認

### 問題: コードが自動入力されない

**解決策:**
1. コンテンツスクリプトがページに注入されているか確認（デベロッパーツールのコンソールをチェック）
2. 入力フィールドが制限されたURL（chrome://など）にないか確認
3. メールに実際に認証コードが含まれているか確認
4. ポップアップのエラーメッセージを確認

### 問題: "認証に失敗しました" エラー

**解決策:**
1. ブラウザのキャッシュとCookieをクリア
2. 拡張機能を削除して再インストール
3. Google Cloud Consoleで OAuth 同意画面が正しく設定されているか確認
4. 必要なスコープ（`gmail.readonly`）が含まれているか確認

## 📝 よくある質問（FAQ）

### Q: この拡張機能は安全ですか？

A: はい。拡張機能は以下の理由で安全です:
- Gmail への読み取り専用アクセスのみ
- すべての処理がローカルで実行
- データを外部サーバーに送信しない
- オープンソースで監査可能

### Q: どのような種類のコードに対応していますか？

A: 4〜8桁の数字および英数字コードに対応しています:
- 数字のみ: 4, 5, 6, 7, 8桁
- 英数字: 6〜8文字

### Q: 認証コードはどこに保存されますか？

A: どこにも保存されません。コードはメールから抽出され、即座にフォームに入力された後、破棄されます。

### Q: 複数のタブで動作しますか？

A: はい。拡張機能は開いているすべてのタブにコードを送信しようとしますが、chrome:// などの制限されたページはスキップされます。

### Q: Gmail 以外のメールサービスに対応していますか？

A: いいえ、現在はGmailのみに対応しています。Gmail API を使用しているためです。
